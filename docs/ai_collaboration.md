# AI Collaboration — Style Superman

> 2026-06-11 第一性原理瘦身：本檔從 259 行的多 AI 組織設計（Codex/Claude Code 交接流程、任務卡模板、RACI 全表）
> 砍到只剩真正載重的規則。歷史版本見 git。agent 的作業入口是 [`../CLAUDE.md`](../CLAUDE.md)，本檔只補它沒有的三件事。

## 1. 角色是帽子，不是產品

「主編」（內容判斷、規格、契約）與「工程」（實作、測試、批次修改）是**職能**，不綁定特定 AI 產品。
同一個 agent 可以在一個任務裡先後戴兩頂帽子，但**換帽要明說**，且一個任務同時只有一頂帽子 accountable。

## 2. 自我審查偏誤控制（本檔最重要的一條）

- 同一個 agent / 同一段對話**不終審自己的產出**。
- 內容判斷類（headline、月報主榜、挑買判斷）終審永遠是人類。
- 工程類大改（架構、契約變更）的 review 應由獨立 session / 獨立 agent 做，
  或至少在 PR 明確標注「未經獨立 review」。

## 3. 誰拍板（最短版）

| 事項 | 拍板者 |
|------|--------|
| 趨勢判讀（炒作 vs 真趨勢）、headline 採用、挑買建議方向 | **人類** |
| 新增來源 / 品牌 / 人物、花錢、對外發布 | **人類**（CLAUDE.md「不應該單獨做」清單） |
| prompts / templates 契約內容 | 主編帽提案，人類確認 |
| 腳本、測試、CI、格式批次整理 | 工程帽自主，CI 綠即可 |

防呆鐵則（不虛構、契約同步、schema 改完跑腳本驗證）已收斂在 [`../CLAUDE.md`](../CLAUDE.md)，不在此重複。
