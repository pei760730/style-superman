# Style Strategy — Style Superman

把情報變成更好的穿搭與更聰明的購買。這份文件定義「我為什麼追、追什麼、用什麼角度看」。

## 1. 對象

唯一對象：**我自己——一個對男裝潮流有興趣、會親手挑單品入手的玩家。**
我想穿得更好、跟上值得跟的潮流，但被資訊量淹沒、也怕亂買踩雷。這套系統替我過濾、翻譯、判斷該不該買。

風格主場：**日系 contemporary lane**（AURALEE / CIOTA / NEAT / COMOLI / A.PRESSE 一帶），重質感、重版型，預算理性；旁及歐美街頭與高端的可借鏡處。

## 2. 情報支柱（我每天在看什麼）

| 支柱 | 佔比 | 說明 | 對應 buy_angle |
|------|:----:|------|----------------|
| 趨勢理解 | 30% | 「這個趨勢是什麼、從哪來、會不會持續」 | — |
| 怎麼穿 / 怎麼搭 | 30% | 「能不能融入我的衣櫥、怎麼搭」 | staple / seasonal |
| 值不值得買 | 20% | 「該入手、該等、還是該跳過」 | upgrade / experiment |
| 單品 / 品牌深掘 | 10% | 版型、布料、價格帶、保值、在哪買 | statement |
| 盤點 / 清單 | 10% | 「本季值得收的幾件」 | — |

刻意保留 20% 給「值不值得買」的冷靜判斷——挑買的價值一半在於避免亂買。

## 3. 個人紀錄方式

不做平台分發。情報與決策落在我每天會看的地方即可：

| 載體 | 角色 |
|------|------|
| `reports/daily/` | 每日情報快照，可長期回看趨勢演化 |
| 週挑 shortlist / brief For Me | 想入手的單品候選，含預算帶與優先度 |
| 衣櫥回饋 | 入手後「實際穿了幾次」，回頭校準挑買判斷 |

一條趨勢 → 看懂它 → 決定買不買 → 入手後回看，最大化單一情報對「我穿得更好」的幫助。

## 4. 調性（給 prompts 用）

- **自信但不裝** — 有觀點，但不裝懂。
- **實用優先** — 每條情報要讓我「看懂 / 想試 / 想買 / 該避開」其中之一。
- **快、準、有態度** — 不講廢話開場，先給重點。
- **誠實** — 不為跟風吹捧不值得的趨勢；該勸退就勸退（「現在別買」也是結論）。

## 5. 成功指標（個人）

（節奏與情報→行動的轉換流程見 [Flow Calendar](flow_calendar.md)，不在此重複。）

| 層級 | 指標 |
|------|------|
| 挑買 | 入手命中率（買了之後實際穿的比例）、不踩雷、cost-per-wear |
| 情報 | brief 命中率（看好的趨勢後續真的紅 / 真的值得入手） |
| 風格 | 衣櫥一致性、穿搭信心 |

這套系統的終極考核：**我是不是比別人早一步看到趨勢、又沒有為此亂買。** 用 `reports/daily/` 的歷史回測這件事。
